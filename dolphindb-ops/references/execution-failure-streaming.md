# 流计算执行错误案例

> 触发: 流表订阅报错、流引擎报错、发布/订阅失败、反序列化失败、流表操作异常

## 规则

### 订阅时报 Can't find the object with name xxx
条件: 报错 `Can't find the object with name xxx`
根因: 订阅流表时目标流表不存在或名称拼写错误
处置: 确认流表已创建（查看共享表列表）；核对流表名称大小写和节点

### 反序列化报错
条件: 二进制反序列化失败、订阅收到脏数据
根因: 发布端 schema 变更后订阅端未同步更新；或网络传输异常
处置: 重新取消并重建订阅；确认两端 schema 一致

### 发布队列深度报警 / 发布阻塞
条件: 发布节点日志中出现"queue depth"警告或发布阻塞
根因: 订阅端消费速度跟不上发布端速率，导致发布队列堆积
处置: 调大 maxPubQueueDepthPerSite；优化订阅端处理逻辑；增加消费并行度

### 引擎已存在
条件: 报错 `The shared object [xxx] already exists`
根因: 重复创建同名流引擎或共享表
处置: 先删除旧的流引擎或释放共享变量，再重建

### enableTableShareAndPersistence 报错
条件: 持久化共享流表操作失败
根因: 表已被共享/持久化过；或参数格式不正确
处置: 确认表是否已共享并持久化（查看共享对象列表）；避免重复操作

### 流表内存占用持续增长
条件: 流表占用内存不断增长，超出预期
根因: 未设置 cacheSize 或值设置过大；无消费者导致数据堆积
处置: 设置合理的 cacheSize（如 100000 行）；确保有活跃订阅消费数据；或定期清理持久化流表

## 验证
- 确认订阅状态正常（查看发布表和订阅 worker 状态）
- 写入测试数据，确认端到端流处理正常
