# 磁盘满 / 空间不足诊断

> 触发: 磁盘满、空间不足、no space、磁盘告警、写入失败、GC disk

## 诊断与清理工具链

1. 查看各分区使用率
2. 查看已知存储目录总大小
3. **钻取目录内部**，按大小排序列出文件和子目录
4. 清理单个文件（默认仅预览，需二次确认后执行）
   - 删除模式（默认）: 删除文件，跳过被进程写入的文件
   - 截断模式: 截断文件为 0 字节（**适用于活跃日志文件**，进程可继续写入，空间立即释放）
   - 路径自动校验必须在节点已知目录范围内
   - 批量清理: 先定位大文件，再逐个清理

## 规则

### 数据目录磁盘满
条件: DolphinDB 数据目录所在分区满
处置:
  1. 查看各分区使用率，确认哪个分区满
  2. 查看已知存储目录总大小，钻取定位大目录/大文件
  3. 预览清理计划，确认后执行清理
  4. 扩容磁盘或迁移数据到新 volume
根因: 排查大目录，关注 volumes、log、redo log；若涉及磁盘损坏需评估 RAID 状态和备份可用性

### Redo Log 占用过大
条件: redo log 目录占用异常大
处置:
  1. 正常关闭节点触发 checkpoint 回收 redo log
  2. 若无法正常关闭，评估是否可删除已 checkpoint 的 redo log
  3. 调整 redoLogPurgeLimit 参数
根因: 检查 redo log 是否正常回收、是否存在大事务未提交

### 日志文件膨胀
条件: DolphinDB 日志文件过大
处置:
  1. 查看日志目录中各文件大小
  2. 对异常大且非活跃的文件：直接删除
  3. 对正在被写入的活跃日志：截断文件为 0 字节（进程可继续写入，空间立即释放）
  4. 将日志级别调回 WARNING/INFO
  5. 配置日志轮转（maxLogSize 参数）
根因: 检查日志级别（是否 DEBUG/TRACE）、是否有刷屏异常

## TID 文件未回收
条件: 分区文件夹里面残留异常数量的，包含_tid_的文件夹
处置:
  1. 扫描分区文件夹是否存在包含_tid_的文件夹
  2. 根据文件夹修改时间排序
  3. 文件夹修改时间过早的比如一个月之前的可以删除
根因: 300.3.1/300.2.7/200.17/200.16.2/200.13.14/200.12.12/200.11.17 之前版本有 bug 不会回收包含_tid_的临时文件夹

### GC 磁盘空间不足
条件: 日志出现 GC disk 相关信息
处置:
  1. 排查长时间运行的查询，取消阻碍 GC 的异常查询
  2. 手动触发 GC
  3. 扩容磁盘空间
根因: GC 进程无法清理过期版本，可能因查询持有旧版本

### 磁盘满导致节点无法启动
条件: 节点因磁盘满启动失败
处置: 先释放磁盘空间（清理日志/临时文件），再启动节点；参考 `startup-failure`

### 磁盘满导致节点崩溃
条件: 写入时磁盘满导致节点 crash
处置: 释放磁盘空间，重启节点检查 redo log 恢复；参考 `crash`

### 分级储存冷盘残留LOG文件定位和清理
条件: 冷盘储存目录的 LOG 文件夹有大量文件残留
处置：找到相关冷盘 LOG 文件夹路径，根据文件夹日期排序，删除较久日期的文件（比如一天前的）。
根因: 软件BUG原因，冷盘 LOG 文件没有回收。

## 验证
- 磁盘各分区使用率回落到安全水位
- 节点状态恢复正常
- 日志无新的写入失败错误
