# 文件堆积 / 残留清理指南

> 触发: 磁盘空间持续增长、持久化流日志堆积、tid 残留文件夹、rollback 异常残留

## 规则

### 持久化流日志文件无法删除导致堆积
条件: 持久化流数据持续堆积并占满磁盘，日志反复出现 `Failed to remove streaming log file`
根因: 历史序号文件缺失，回收逻辑反复检查同一缺失文件而卡住，后续文件无法正常回收
处置:
1. 查看告警日志，提取失败的 `fileName` 与序号信息
2. 检查当前持久化文件序列，定位缺失段
3. 按序号补齐缺失文件（确保链路连续）
4. 观察回收流程恢复推进

### tid 残留文件夹定位与清理
条件: 存储目录出现大量 `_tid_` 文件夹残留（rollback 异常导致）
处置: 使用 `getTabletsMeta` 遍历分区路径，筛选含 `_tid_` 的子目录；核对路径是否为残留目录，结合 `lastAccessed/lastModified` 判断是否仍在使用，必要时抽样备份后删除
根因: 如残留持续增长，排查 rollback 根因，300.3.1/300.2.7/200.17/200.16.2/200.13.14/200.12.12/200.11.17 之前版本有 bug 不会回收包含_tid_的临时文件夹

### 分级储存冷盘残留LOG文件定位和清理
条件: 冷盘储存目录的 LOG 文件夹有大量文件残留
处置：找到相关冷盘 LOG 文件夹路径，根据文件夹日期排序，删除较久日期的文件（比如一天前的）。
根因: 软件BUG原因，冷盘 LOG 文件没有回收。

## 验证
- 告警消失 + 持久化流文件开始正常回收
- 磁盘占用趋势下降
- 无新的残留目录产生
