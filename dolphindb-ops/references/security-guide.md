# 权限与安全问题诊断

> 触发: 用户无法访问数据库/表、权限配置错误、权限冲突、敏感数据访问控制问题

## 规则

### 用户无法读取表
条件: 日志出现 `Not granted to read table` 或用户报告无法查询表
处置:
1. 检查用户权限
2. 检查用户所属组
3. 检查各组权限
4. 确认用户或其所属组是否有 TABLE_READ 或 DB_READ 权限
根因: 用户未被授予读取权限，或权限被 deny

### 用户无法写入表
条件: 日志出现 `Not granted to write table` 或用户报告无法写入数据
处置:
1. 检查用户权限
2. 检查用户所属组
3. 确认用户是否有 TABLE_WRITE/TABLE_INSERT/DB_WRITE/DB_INSERT 权限
根因: 用户未被授予写入权限

### 用户无法创建数据库
条件: 日志出现 `Not granted to create or delete databases`
处置:
1. 检查用户权限
2. 确认用户是否有 DB_OWNER 权限
根因: 用户未被授予 DB_OWNER 权限

### 用户无法创建表
条件: 日志出现 `Not granted to create table`
处置:
1. 检查用户权限
2. 确认用户是否有 DBOBJ_CREATE 或 DB_OWNER 权限
根因: 用户未被授予建表权限

### 用户无法删除表
条件: 日志出现 `Not granted to delete table`
处置:
1. 检查用户权限
2. 确认用户是否有 DBOBJ_DELETE 或 DB_OWNER 权限
根因: 用户未被授予删除表权限

### 函数视图执行失败
条件: 日志出现 `Not granted to execute function view`
处置:
1. 查看所有函数视图列表
2. 检查用户是否有 VIEW_EXEC 权限
根因: 用户未被授予函数视图执行权限

### 权限冲突
条件: 用户在某个组被 grant 权限，在另一个组被 deny 权限
处置:
1. 查看用户所属的所有组
2. 逐个检查各组的权限
3. 根据权限确定规则：任一组 deny 则最终 deny
根因: 多组权限冲突，deny 优先级高于 allow

### 管理员权限不足
条件: 管理员无法执行某些管理操作（如查看所有用户、修改权限）
处置:
1. 确认当前登录用户
2. 查看用户列表，确认是否为管理员
根因: 当前用户不是管理员或超级管理员

### 无法查看其他用户权限
条件: 查看其他用户权限时失败
处置:
1. 确认当前登录用户
2. 确认当前用户是否为管理员
根因: 只有管理员可以查看其他用户的权限，普通用户只能查看自己的权限

### 组成员管理失败
条件: 无法添加/删除组成员
处置:
1. 确认当前登录用户
2. 查看组成员列表
3. 确认当前用户是否为管理员
根因: 只有管理员可以管理组成员

### 权限审计需求
条件: 需要定期审计权限分配情况
处置:
1. 获取所有用户列表
2. 获取所有组列表
3. 逐个检查用户权限
4. 逐个检查组权限
5. 检查函数视图权限
根因: 定期安全审计要求

### 数据节点权限函数不可用
条件: 在数据节点执行权限函数失败
处置:
1. 检查 DolphinDB 版本是否为 2.00.9/1.30.21 或更高版本
2. 如果版本较低，切换到控制节点执行权限函数
根因: 旧版本只能在控制节点执行权限函数

## 验证
- 确认权限已正确配置
- 确认用户组关系正确
- 测试用户是否能正常访问资源
- 检查日志中是否还有权限相关错误

## 权限类型说明

### 数据库权限
- `DB_READ`: 读取数据库中所有表
- `DB_INSERT`: 向数据库中所有表插入数据
- `DB_UPDATE`: 更新数据库中所有表
- `DB_DELETE`: 删除数据库中所有表的数据
- `DB_WRITE`: 包含 INSERT、UPDATE、DELETE
- `DB_OWNER`: 数据库所有者，包含所有权限

### 表权限
- `TABLE_READ`: 读取表
- `TABLE_INSERT`: 向表插入数据
- `TABLE_UPDATE`: 更新表
- `TABLE_DELETE`: 删除表中的数据
- `TABLE_WRITE`: 包含 INSERT、UPDATE、DELETE

### 对象权限
- `DBOBJ_CREATE`: 创建数据库对象（表、分区等）
- `DBOBJ_DELETE`: 删除数据库对象

### 函数视图权限
- `VIEW_EXEC`: 执行函数视图

### 脚本权限
- `SCRIPT_EXEC`: 执行脚本

### 测试权限
- `TEST_EXEC`: 执行测试

## 注意事项

1. **权限优先级**: deny 优先于 allow，任一组 deny 则最终 deny
2. **管理员权限**: 大部分权限查询函数需要管理员权限
3. **版本兼容性**: 2.00.9/1.30.21 及以上版本支持在数据节点执行权限函数
4. **权限继承**: 用户继承其所属组的所有权限
5. **超级管理员**: admin 用户拥有所有权限，无法被限制
