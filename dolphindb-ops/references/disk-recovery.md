# 磁盘损坏与数据恢复操作指南

> 触发: 磁盘损坏、文件丢失、副本恢复、磁盘容量调整、换盘、数据平衡

## 规则

### 双副本单盘损坏恢复
条件: 数据节点某块非首 volume 磁盘损坏，分区不可用（`chunkMetaDir` 不在损坏盘上）
处置: 查受影响 chunkIds → 删除损坏副本 → 确认 replicaCount=1 → 从另一节点复制副本恢复

### 坏盘物理更换与恢复
条件: 数据节点磁盘硬件故障
处置: 关集群 → 换盘挂载 → 校验副本状态 → 批量删除坏盘元数据 → 从其他节点逐一复制副本恢复

### Symbol 文件损坏
条件: 报错 `The symbol column xxxx is corrupted` 或 `Invalid symbol file`
根因: `.col` / `chunk.dict` 文件损坏
处置: 报错路径定位 chunkId → 验证另一副本健康 → 删除损坏副本 → 从健康节点复制副本

### Level File 丢失
条件: 报错 `File [...] not exists`
根因: 偶发文件损坏或并发 rollback 导致 level file 丢失
处置: 查询所有数据节点 chunk 状态定位 chunkId → 从好节点复制副本覆盖恢复

### 新增磁盘后手动数据平衡
条件: 新盘未被均衡使用，内置数据平衡功能效果异常
处置: 统计各 volume 分区数 → 计算目标均值 → 从旧盘提取超出部分 → 轮转迁移副本到新盘

### 3 副本调整为 2 副本
条件: 磁盘容量紧张，需降低冗余
处置: 修改 dfsReplicationFactor=2 重启 → 分批删除多余副本 → 校验 replicaCount=2 → 可选重新均衡各节点分区

## 关键操作风险提示
- `deleteReplicas` / `copyReplicas1` 是破坏性操作，执行前确认影响范围
- 换盘操作需在维护窗口内，确保集群停写入
- 操作前**必须备份**元数据目录
- 单副本场景下磁盘损坏可能导致不可恢复的数据丢失

## 验证
- 副本状态恢复正常（replicaCount 一致）
- 分区数据可正常读写
- 磁盘空间分布符合预期
