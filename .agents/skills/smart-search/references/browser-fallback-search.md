# 浏览器回退搜索策略

当 OpenCLI 适配器不足、且主要搜索引擎全部被验证码拦截时的回退方案。

## 适用场景

- OpenCLI 没有对应站点的适配器
- Google / Bing / Baidu / DuckDuckGo 全部被 CAPTCHA 拦截
- 需要从中文网站提取特定结构化数据（价格、日期、场馆等）

## 可用回退搜索引擎

| 引擎 | URL | 特点 |
|------|-----|------|
| **360 搜索** | `so.com` | 反爬相对宽松，搜索结果片段（snippet）包含丰富结构化数据如价格、时间、评分。适合快速获取概览。 |
| **新浪搜索** | `search.sina.com.cn` | 综合搜索，包含新闻、视频、微博等内容，不容易被拦截。可指定 `c=news` 过滤新闻或 `time=week` 限定时间范围。 |
| **搜狗搜索** | `sogou.com` | 中文搜索覆盖较好，但有时会超时。 |

## 搜索策略

### 1. 先用 360 搜索获取概览

360 搜索的结果摘要含有丰富的结构化信息。搜索时用精确短语：

```
so.com/s?q=精确主题+限定词1+限定词2
```

例如：`杨丞琳+房间里的大象+上海+380+580+780+980+1280`

直接在搜索词中包含可能的价格/参数，360 的摘要能直接暴露匹配内容。

### 2. 用新浪搜索获取实时内容

新浪搜索对实时新闻和社媒内容的收录较好。支持时间范围限定：

```
search.sina.com.cn/search?q=关键词&range=all&c=news&time=custom&stime=2026-05-01&etime=2026-05-24
```

### 3. 直接导航到垂直站点

当搜索引擎结果不足时，直接访问行业垂直站点，例如：

| 场景 | 站点 |
|------|------|
| 演出票务 | `piaoniu.com`、`damai.cn`、`dahepiao.com` |
| 酒店 | `ctrip.com`、`booking.com` |

注意：已过期的演出在 piaoniu.com 上只显示"已过期"不显示具体票价，但可以从搜索引擎的摘要中提取到价格信息。

### 4. 社交媒体交叉验证

- **微博**：通过 `opencli weibo feed` 获取实时讨论。微博上有大量演出相关帖子，能确认场馆、座位争议、票价等细节。
- **小红书**：通过 `opencli xiaohongshu search`。

## 定价信息提取技巧

对于中文演出票价查询，常见的定价模式为多档位：

```
380 / 580 / 780 / 980 / 1080 / 1280（元）
```

直接在搜索词中包含这些常见价格区间，360 搜索的摘要很可能会直接显示匹配到的价格。

## 已知限制

- 这些回退搜索引擎不支持程序化/API 访问，必须用 browser 工具
- 360 搜索对大流量请求仍可能触发验证码（概率比百度低）
- 新浪搜索的结果质量依赖于搜索词精确度
- 对于已过期事件，垂直站点（piaoniu、damai）的价格详情页会被隐藏
