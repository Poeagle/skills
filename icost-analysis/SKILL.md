# iCost 个人消费分析系统

## 核心逻辑：预算对比法

不再按「必要/正常/浪费」分类，而是：

> **设定预算 → 对比实际 → 计算超支**

### 用户设定的预算标准

| 餐别 | 预算上限 | 工作日 | 周末 |
|------|---------|--------|------|
| 早餐 | ¥5 | 自费 ✓ | 自费 ✓ |
| 午餐 | ¥10 | **公司管饭，免费** 🏢 | 自费 ✓ |
| 晚餐 | ¥10 | 自费 ✓ | 自费 ✓ |
| **每天自费合计** | | **¥15**（早+晚） | **¥25**（早+午+晚） |

**计算逻辑（report.py 中的 budget_per_day）：**
- 工作日（Mon-Fri）：`return 15`（早¥5 + 晚¥10，午餐公司免费）
- 周末（Sat-Sun）：`return 25`（早¥5 + 午¥10 + 晚¥10）
- 早餐预算已含在总预算中，但 iCost 无早餐单独记录，追踪时只对比午+晚实际支出

**⚠️ 踩坑历史：**
- ❌ 一开始按所有外食做「必要/正常/浪费」分类 → 被用户否定（数字对不上、主观判断不准）
- ❌ 用了 ¥15/顿的自炊标准算节省 → 用户说没理解「工作日午餐免费」的前提
- ❌ 忘了加早餐预算 → 用户暴怒
- ✅ 最终方案：直接设预算线，对比实际支出，算超出金额

## 报告逻辑

### 上周
- 消费总览：按一级分类拆分
- 伙食预算对比：实际 vs 预算（工作日¥15、周末¥25）→ 超出/节余
- 列出单餐超¥10的明细项

### 本月
- 总消费 + 日均
- 伙食超支总额（实际 - 天数×预算）
  - 预算 = 工作日数×¥15 + 周末天数×¥25
- 每周拆分对比（按自然周 Mon-Sun）
  - 周范围 M/D-D  实际¥XXX  预算¥XXX  🔺超/✅省

### 零食饮料
- 单独列出，视为「不在预算内的额外支出」

### 总结
- 伙食控制：省 ¥XXX
- 零食不买：省 ¥XXX
- 理想月支出

```
📋 消费周报
────────────────────
📆 上周 M/D-M/D
消费 ¥XXX（餐饮¥XXX · 日用¥XXX · 软件¥XXX）
伙食 ¥XXX，预算 ¥XXX，超出 ¥XXX
超标项：M/D¥XX备注 · M/D¥XX备注

────────────────────
📆 本月 1-D（X 天）
总消费 ¥XXXX（¥XXX/天）
伙食超支 ¥XXX（实际 ¥XXX - 预算 ¥XXX）

────────────────────
📊 每周伙食对比（工作日¥15/天 · 周末¥25/天 · 含早¥5）
第1周 5/1-5/7  实际¥55  预算¥125  ✅省¥70
第2周 5/8-5/14  实际¥246  预算¥125  🔺超¥121
...

────────────────────
🍿 零食饮料 ¥XXX — 不在预算内，额外支出

────────────────────
✅ 如果零浪费：原本 ¥XXXX/月
  伙食控制在预算内 → 省 ¥XXX
  不买零食 → 省 ¥XXX
  每月只需 ¥XXXX（¥XXX/天）

────────────────────
📌 资产更新了没？
```

### 格式死律
- 分隔线用 `─` x 20
- 数字用 ¥ 前缀，不要加"元"字
- 超标项用 `·` 分隔，不要换行（微信一条消息不要太长）
- 每周都要有上周数据，不能只报本月
- 末尾必须加资产提醒

## 工作流

### 每周用户操作
周日从 iCost 导出增量 xlsx（上周新记录），保持 iCost 默认命名（iCost_YYYYMMDD*.xlsx），放入 `~/records/finance/`

### 周一 07:00 自动流程
1. **扫描新 xlsx** — 对比 processed.log，有则调用 ingest.py 转换
2. **漏传检测** — 无新文件且最后记录不在上周 → 微信提醒
3. **分析** — 读年度 csv，按预算规则计算
4. **推送** — 按固定格式生成报告，发微信

### 手动查询模式
用户直接问时：
1. 读对应年份 csv
2. 直接出结论：总支出 + 伙食超支 + 零浪费月支出
3. 不列原始数据流水账

## 脚本说明

### ingest.py
- 读取 xlsx 的 `收支账单` sheet
- 仅保留 类型 == "支出" 的记录
- 丢弃图片相关列
- 按日期拆分到年度 csv，去重后追加
- 写入 processed.log，删除原始 xlsx

### report.py
- 读年度 csv
- 按预算规则计算各维度
- 每周拆分（从月初开始按自然周划分）
- 生成报告 → print + send_message（推送 QQ）
- 微信推送失败时打印到终端作为 fallback

## cron 配置

```
0 7 * * 1 python3 ~/.claude/skills/icost-analysis/scripts/report.py 2>> ~/records/finance/crontab.err.log
```

由 Hermes cronjob 工具设定，每周一 07:00 执行。
