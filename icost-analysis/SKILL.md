## 核心死律（所有模式强制执行）

1. **禁止语义化日期**：禁止"11月初"、"上周末"等模糊描述。必须回显 `YYYY/MM/DD HH:MM:SS` 原始时间戳。
2. **脚本驱动输出**：严禁手工整理账单列表。必须编写并运行 Python 脚本，由脚本输出完整的、带行号的数据块。助理只能 1:1 搬运脚本输出。
3. **原始备注原则**：备注内容 100% 还原，不做摘要、翻译或美化。
4. **行号溯源**：所有明细必须附带 csv 行号或 Excel 行号。
5. **禁止依赖记忆**：涉及金额的查询必须重新读文件，禁止使用对话历史中的数字。

## 模式 B：自动周报流程（完整步骤）

### 步骤 1：扫描新文件
- 遍历 `/Users/ymchen/records/finance/` 下的 `*.xlsx` 文件
- 与 `processed.log` 对比，跳过已处理的文件名
- 没有新文件 → 跳到步骤 3（漏传检测）
- 有新文件 → 走步骤 2

### 步骤 2：导入新数据
对每个未处理的 xlsx：
1. 调用 `ingest.py` 读取 xlsx 的 `收支账单` sheet
2. 丢弃图片相关列（图片链接1-3、图片附件1-3）
3. 按日期拆分到对应年份的 csv（追加，不覆盖）
4. 将文件名写入 `processed.log`
5. 删除原始 xlsx 文件

### 步骤 3：漏传检测
- 如果步骤 1 没有新文件：
  - 读取当年 csv 的最后一条记录日期
  - 如果该日期不在上周范围（周一~周日）内 → 通过微信提醒用户漏传
  - 如果该日期在上周范围内 → 正常，继续

### 步骤 4：消费分析
调用 `report.py`，生成三个维度的分析：

**上周支出**（上周一 ~ 上周日）：
- 总支出
- 按一级分类汇总（金额 + 占比 + 笔数）
- 环比上上周（金额变化 + 百分比）
- 异常大额条目标记（单笔超过同类均值 2σ）

**本月累计**（本月1日 ~ 昨天）：
- 总支出
- 按一级分类汇总
- 环比上月同期

**本年累计**（本年1月1日 ~ 昨天）：
- 总支出
- 日均支出
- 年化推算
- 按 FIRE 目标（150万，月支出目标 ~¥4,000）评估进度

### 步骤 5：生成建议
基于数据分析，给出可执行的建议：
- 哪些分类有压缩空间（环比上涨的分类）
- 哪些单笔消费值得反思（异常大额）
- 本月剩余天数的预算建议
