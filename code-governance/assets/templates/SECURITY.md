# Security Policy

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 1.x  | ✅ 活跃支持 |
| <1.0 | ❌ 停止支持 |

## 报告漏洞

我们非常重视安全。如果发现安全漏洞，请通过以下方式报告：

- **Email**: security@{{project_name}}.com
- **预期回复时间**: 48 小时内

请**不要**在公开的 issue 中报告安全漏洞。

## 安全实践

### 依赖安全
- 所有依赖定期通过 Dependabot / Renovate 自动更新
- 每次 CI 运行时执行 `npm audit` 或等效检查
- 禁止引入已知有安全漏洞的依赖版本

### 敏感信息
- 敏感信息（API key、token、密码）通过环境变量注入
- `.env.example` 作为模板提交到仓库
- `.env` 文件永不提交（已在 .gitignore 中排除）
- CI secrets 通过 CI 平台的 secrets 管理功能配置

### 输入验证
- 所有外部输入（API 请求、文件上传、用户输入）必须在边界验证
- 使用 Zod schema 或等效方案进行运行时类型验证
- 输出编码防止 XSS 攻击

### 生产安全
- 生产环境日志不能输出敏感字段
- 使用 HTTPS 传输数据
- 遵循最小权限原则配置访问控制

## 安全更新

安全相关的修复将优先处理，并在修复后尽快发布补丁版本。
