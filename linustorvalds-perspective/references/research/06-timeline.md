# Linus Torvalds 完整时间线

## 一、童年与早期编程启蒙（1969-1987）

| 年份 | 事件 | 详情 | 确定性 |
|------|------|------|--------|
| 1969-12-28 | **出生** | 出生于芬兰赫尔辛基，父亲 Nils Torvalds 和母亲 Anna Torvalds 均为记者。他的名字取自化学家 Linus Pauling 和《花生漫画》角色 Linus van Pelt。家族属于芬兰瑞典语少数民族 | [✓] |
| 1970s | **童年** | 外祖父 Leo Törnqvist 是统计学家（赫尔辛基大学教授），舅舅们任职于计算机行业。父母是 1960 年代赫尔辛基大学的校园激进分子 | [✓] |
| 1981 | **首次接触计算机** | 11 岁时外祖父购入 Commodore VIC-20。Linus 开始用 BASIC 编程，随后深入 6502 CPU 机器码。阅读了大量 Commodore 编程书籍 | [✓] |
| 1980s 中 | **Sinclair QL 时期** | 购入 Sinclair QL（Quantum Leap），对其高度不满但因此深入底层。自行编写了汇编器、图形库、游戏（含 Pac-Man 克隆 "Cool Man"）。掌握 Motorola 68000 汇编 | [✓] |

> 💡 **思想萌芽**：早年对受限硬件的底层编程经历，塑造了他后来对系统编程中"控制权"的执着——必须能触及硬件每一比特。

## 二、大学与 Linux 诞生前奏（1988-1991）

| 年份 | 事件 | 详情 | 确定性 |
|------|------|------|--------|
| 1988 | **进入赫尔辛基大学** | 主修计算机科学，加入 NODES 研究组 | [✓] |
| 1989 夏 | **芬兰兵役** | 加入芬兰海军新芬兰旅（Nyland Brigade），完成 11 个月军官培训项目，获少尉军衔（炮兵观察员） | [✓] |

| 1991 夏 | **项目启动** | 开始开发一个"只是业余爱好"的操作系统内核，最初使用 Intel 386 保护模式下的任务切换，逐步实现文件系统和驱动 | [✓] |

## 三、Linux 诞生与早期发展（1991-1996）

| 年份 | 事件 | 详情 | 确定性 |
|------|------|------|--------|
| 1991-08-25 | **著名的 Usenet 公告** | 在 comp.os.minix 新闻组发帖："Hello everybody out there using minix — I'm doing a (free) operating system (just a hobby, won't be big and professional like gnu) for 386(486) AT clones." 这成为 Linux 的"出生证明" | [✓] |
| 1991-09 | **0.01 版** | 内部版本 0.01，仅有基本内核，无 shell，无网络。Linus 最初拟命名为 **Freax**（free + freak + X），但 FTP 管理员 Ari Lemmke 擅自创建了名为 *linux* 的目录 | [✓] |
| 1991-10-05 | **0.02 版（首个公开版）** | 第一个公开版本，可以运行 bash、gcc 和 gnu-make。约 100 行汇编 + 约 10000 行 C | [✓] |
| 1991-12 | **0.11 版（自编译里程碑）** | 首个能在 Linux 自身环境下编译的内核版本，标志着 Linux 实现自举（self-hosting） | [✓] |
| 1992-02 | **切换至 GPL 许可证** | 在 0.12 版中将原有禁止商业再分发的自定义许可证切换为 **GNU GPLv2**。这一决定受到 Richard Stallman 演讲启发以及社区贡献者的压力推动 | [✓] |
| 1992 | **Tanenbaum-Torvalds 辩论** | Andrew Tanenbaum 在 Usenet 上发表"Linux is obsolete"的著名批评，认为微内核架构优于宏内核。Linus 与之展开经典辩论——**这是 Linus 工程哲学的重要成型时刻** | [✓] |
| 1993 | **内核走向成熟** | 社区贡献者快速增加，大量志愿者贡献驱动和子系统。Linus 扮演"仁慈独裁者"（BDFL）角色 | [✓] |
| 1994-03-14 | **Linux 1.0.0** | 第一个产品级版本，含 176,250 行代码。标志 Linux 不再是"业余项目" | [✓] |
| 1995 | **1.2.0 发布** | 支持更多架构（Alpha、MIPS、SPARC），驱动数量大幅增加 | [✓] |
| 1996-06-09 | **Linux 2.0.0** | 引入对称多处理（SMP）支持，大幅扩展处理器架构支持。代码量突破百万行 | [✓] |
| 1996 底 | **Tux 吉祥物诞生** | Larry Ewing 使用 GIMP 创作了 Linux 企鹅吉祥物 **Tux**。Linus 选择企鹅是因为"被企鹅咬过"——据传他在澳大利亚动物园被一只小企鹅咬过 | [✓] |

> 💡 **思想转折点：GPL 切换**。从自定许可证到 GPLv2 的决定，是 Linus 对开源协作模式的第一次深刻认知变化——他意识到**共同创作需要共享的法律框架**。

## 四、Transmeta 时期与 Linux 起飞（1997-2003）

| 年份 | 事件 | 详情 | 确定性 |
|------|------|------|--------|
| 1997-02 | **加入 Transmeta** | 移居美国加州圣克拉拉，任职于低功耗 x86 兼容处理器公司 Transmeta。具体角色：负责代码生成质量（code morphing software）。工作不直接等于 Linux 开发，但给了他稳定的收入来源 | [✓] |
| 1997 | **获硕士学位** | 赫尔辛基大学授予计算机科学硕士学位（Laudatur 等级） | [✓] |
| 1998 | **EFF Pioneer Award** | 电子前哨基金会授予先锋奖 | [✓] |
| 1999 | **股票期权事件** | Red Hat 和 VA Linux 上市，Linus 持有的股票期权价值约 2000 万美元，实现财务自由 | [✓] |
| 1999 | **MIT Technology Review TR100** | 入选麻省理工科技评论百大青年创新者 | [✓] |
| 2000 | **Steve Jobs 接洽** | 乔布斯邀请 Linus 加入 Apple，停止 Linux 开发、为 macOS 工作。Linus 拒绝，理由是 Mach 内核与 Linux 差异过大 | [✓] |
| 2000 | **Time 100 人物** | 被《时代》周刊列入 20 世纪百大最重要人物（第 17 位）及"世纪人物" | [✓] |
| 2001 | **Takeda Award** | 与 Richard Stallman、Ken Sakamura 共享武田奖（社会/经济福祉类） | [✓] |
| 2001 | **《Just for Fun》出版** | 自传《只是好玩：一个偶然革命者的故事》出版，合著者为 David Diamond | [✓] |
| 2001-01 | **Linux 2.4.0** | 引入 USB、PC Cards、ISA PnP、Pentium 4 和 Itanium 支持 | [✓] |
| 2002 | **BitKeeper 采用** | Linux 内核开发开始使用 BitKeeper（分布式版本控制系统），大幅提升协作效率 | [✓] |
| 2003-06 | **离开 Transmeta** | 因 Transmeta 业务下滑，Linus 离开公司。加入 OSDL（开源开发实验室）成为全职 Linux 开发者 | [✓] |
| 2003-12 | **Linux 2.6.0** | 重大里程碑：整合 μClinux、ALSA、SELinux、NPTL、内核抢占、16TB 文件系统支持 | [✓] |

> 💡 **思想转折点：全职开源**。加入 OSDL 意味着 Linus 从"有公司赞助的业余项目维护者"转变为**全职开源协调者**。这对他的身份认同产生了深远影响——Linux 不再是副业，而是他的全部职业。

## 五、Git 创建与 Oregon 时代（2004-2010）

| 年份 | 事件 | 详情 | 确定性 |
|------|------|------|--------|
| 2004-06 | **迁居俄勒冈** | 移居俄勒冈州波特兰郊区的 Dunthorpe，靠近 OSDL 总部（Beaverton）。结束硅谷生活 | [✓] |
| 2005-04-03 | **Git 项目启动** | BitKeeper 所有者 Larry McVoy 因 Andrew Tridgell 逆向工程 BitKeeper 协议（开发 SourcePuller），撤销了 Linux 项目的免费许可证。Linus 当日在内核邮件列表做出激烈回应，随后开始编写自己的 VCS | [✓] |
| 2005-04-06 | **Git 公开宣布** | Linus 在 LKML 上宣布 Git 项目。他设定了严苛性能目标：合并操作必须在 **3 秒内** 完成。以 CVS 为反面教材："如有疑问，做相反的决定" | [✓] |
| 2005-04-07 | **首个 Git 提交** | "Initial revision of 'git', the information manager from hell"——项目当天即实现自托管 | [✓] |
| 2005-04-29 | **性能达标** | Git 达到 6.7 patches/秒的合并速度，满足 Linus 的设计目标 | [✓] |
| 2005-06-16 | **Linux 2.6.12 用 Git 发布** | Linux 内核首次通过 Git 发布，标志着新工具通过"生产考验" | [✓] |
| 2005-07-26 | **移交 Git 维护权** | Linus 将 Git 的日常维护移交给 Junio Hamano。**这是重要的管理智慧**——懂得何时放手 | [✓] |
| 2005-12-21 | **Git 1.0** | Junio Hamano 发布 Git 1.0 版本 | [✓] |
| 2006 | **OSDL → Linux Foundation** | OSDL 与 Free Standards Group 合并为 Linux Foundation，Linus 继续在其资助下工作 | [✓] |
| 2006 | **Time 革命英雄** | 《时代》欧洲版将 Linus 评为过去 60 年革命英雄之一 | [✓] |
| 2007 | **Google 公开称赞** | Google 工程师 Andrew Morton（内核维护者之一）在公开场合评论 Linux 内核的质量和开发流程 | [✓] |
| 2008 | **计算机历史博物馆** | 入选计算机历史博物馆荣誉院士 | [✓] |
| 2008 | **使用 Fedora** | 公开表示使用 Fedora Linux（因当时需要 PowerPC 支持） | [✓] |
| 2010 | **C&C Prize** | 获 NEC C&C 奖 | [✓] |
| 2010 | **成为美国公民** | 成为美国公民并登记投票。自称不认同任何美国政党："我自尊心太强，不愿与其中任何一个扯上关系" | [✓] |

> 💡 **思想转折点：Git 设计哲学**。创建 Git 时，Linus 展示了他的**系统工程思维**：从底层需求（不是"我要做个版本管理系统"而是"我需要跟踪树状结构的变化"）出发，设计出内容寻址文件系统而非传统的 SCM。Git 的诞生也体现了他对"自由"的新理解——工具自主权是项目独立的基石。

## 六、盛名之下与持续贡献（2011-2017）

| 年份 | 事件 | 详情 | 确定性 |
|------|------|------|--------|
| 2011 春 | **Linux 3.0.0** | 为纪念 Linux 20 周年发布。Linus 直言大变化"什么也没有，绝对没有"——仅是编号变动。同时加了 uname26 兼容内核 | [✓] |

| 2012-06-13 | **Millennium Technology Prize** | 与山中伸弥（iPS 细胞发现者）共享 **Millennium Technology Prize**（被称为"科技界诺贝尔奖"），奖金 120 万欧元 | [✓] |

| 2012-12 | **i386 支持移除** | Linux 3.7 移除了对 Intel 80386 处理器的支持，Linus 称"为减少复杂性和维护负担" | [✓] |
| 2014 | **IEEE Computer Society Computer Pioneer Award** | 获 IEEE 计算机先驱奖 | [✓] |
| 2016 底 | **Linux 4.9** | 这是当时最大的内核版本之一，包含大量新硬件支持和优化 | [✓] |
| 2017 | **SUSE 发布歌曲 "Linus Said"** | 关于 Linux 起源的纪念歌曲，反映了 Linus 在开源社区的文化影响力 | [✓] |

> 💡 **思想转折点：自我认知矛盾**。这一时期 Linus 多次公开承认自己"不是个友善的人"，但同时坚持技术讨论"应当直截了当"。他意识到自己的沟通风格是双刃剑——提高了效率但也排斥了参与者。

## 七、2018 年休假与沟通风格改革（2018-2019）

| 年份 | 事件 | 详情 | 确定性 |
|------|------|------|--------|
| 2018-09-16 | **行为准则更替** | Linux 内核的 *Code of Conflict* 被替换为基于 Contributor Covenant 的 *Code of Conduct*。Linus 在 4.19-rc4 发布说明中公开道歉 | [✓] |
| 2018-09-16 | **公开道歉与休假声明** | Linus 写道："我很抱歉。过去一段时间我一直在情绪上不专业……我需要休息一下，学习如何理解他人的情感并做出恰当回应" | [✓] |
| 2018-09 — 2018-10 | **休假期** | 离开内核维护岗位数周。《纽约客》在此时对其进行了批评性报道。Linus 参加了情绪管理辅导 | [✓] |
| 2018-10-22 | **回归** | Linux 4.19 发布后 Linus 正式回归，承诺改进沟通方式 | [✓] |
| 2019-03 | **Linux 5.0** | 编号从 4.x 跳到 5.0，Linus 开玩笑说"4.x 的数字大到手指脚趾数不过来了" | [✓] |
| 2019 | **Great Immigrants Award** | 卡内基基金会颁发"伟大移民奖" | [✓] |

> 💡 **思想转折点：核心人格重塑**。2018 年事件是 Linus 人生中**最重要的非技术转折点**。他首次公开承认技术卓越不能替代人际尊重，并在他人帮助下有意识地调整行为模式。这标志着他从"只认代码质量"到"也认社区健康"的转变。

## 十、重大思想转折点汇总

下表列出了 Linus Torvalds 在不同议题上可观察到的思想演变节点。

| 序号 | 领域 | 时间窗口 | 方向转变 | 触发因素 | 确定性 |
|------|------|----------|----------|----------|--------|
| T1 | **许可证选择** | 1991 → 1992 | 自定义许可 → GPLv2 | Stallman 演讲 + 社区压力 | [✓] |
| T2 | **内核架构哲学** | 1992 | 形成"宏内核优于微内核"立场 | Tanenbaum 辩论 | [✓] |
| T3 | **开源商业化接受** | 1999 → 2001 | 从"纯粹理想"到接受"公司参与有益" | Red Hat/VA Linux IPO；亲身获利 | [✓] |
| T4 | **分布式版本控制** | 2002 → 2005 | 从不信赖 DVCS → BitKeeper → 亲手创造 Git | BitKeeper 协议问题 | [✓] |

## 十一、关键思想领域详解

### 11.1 对 C++ 的态度（T6）

Linus 在 2007 年 LKML 上发表了著名的反 C++ rant，核心论点：

- C++ 在底层系统编程中引入了过多隐性开销（隐式构造、异常处理、RTTI）
- C++ 的"语言复杂性膨胀"导致即使顶级工程师也无法掌握全部特性
- 他认为 C 的"极简 + 显式"模式更适合内核开发
- 不过他的反对是针对**内核开发场景**，并非全盘否定 C++ 在其他领域的适用性

> 截至 2026 年，Linus 未在公开场合修正过这一立场。他对 Rust 的接纳源于 Rust 在**内存安全方面的确定性提升**，而非对 C++ 批评的撤回。

### 11.2 对安全/隐私的看法变化

- **早期（1990s）**：安全不是首要关注点——Linux 最初的设计假设是"用户彼此信任"
- **中期（2000s）**：SELinux 等安全模块合并，Linus 趋于实用主义，倾向于模块化的安全实现
- **近期（2020s）**：Rust 的合入表明 Linus 认识到**内存安全是内核目前最大的安全威胁之一**。但他仍强调安全不能以牺牲用户选择权为代价

### 11.3 对开源商业模式的理解演变

| 阶段 | 时期 | 核心认识 |
|------|------|----------|
| 学生理想主义 | 1991-1993 | 开源是道德选择，商业是潜在威胁 |
| 实用开放 | 1994-1998 | 接受公司参与（提供驱动和资金），但保持警惕 |
| 财务自由与反思 | 1999-2003 | Red Hat / VA Linux IPO 后意识到开源可以产生巨额合法财富 |
| 制度化 | 2004-2012 | OSDL/Linux Foundation 模式确立：保持中立的同时让大公司资助核心基础设施开发 |
| 成熟共识 | 2013-至今 | 开源 + 企业赞助 + 生态服务是可持续模式；对 monetziation 持务实态度 |
